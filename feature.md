# ⚡ 피카츄 배구 개발 Feature List

## Phase 1: 개발 환경 및 기본 웹 서빙 구축
* [ ] 1. Flask 기본 서버 개발 및 `index.html` 렌더링 라우팅 구현 ← requirement 기술 스택
* [ ] 2. HTML5 `<canvas>` 태그 배치 및 CSS를 활용한 화면 중앙 정렬 스타일링 ← requirement 디자인 요구
* [ ] 3. JavaScript 게임 루프(`requestAnimationFrame`) 설정 및 동작 확인용 콘솔 로그 출력 ← requirement 필수 기능

## Phase 2: 초기 화면 및 UI 구현
* [ ] 4. HTML/CSS로 Canvas 위에 겹쳐 보일 '게임 시작' 버튼 및 메인 타이틀 레이어 구현 ← requirement 필수 기능
* [ ] 5. 게임 시작 버튼 클릭 시 메인 레이어가 숨겨지고 게임 루프가 활성화되는 이벤트 구현 ← requirement 필수 기능
* [ ] 6. Canvas 상단에 양측 플레이어 점수(0 : 0)를 그리는 UI 렌더링 함수 작성 ← requirement 디자인 요구

## Phase 3: 게임 월드 및 오브젝트 기본 렌더링
* [ ] 7. Canvas 바닥면 경계 설정 및 화면 중앙 하단에 높이 1/3 짜리 직사각형 네트 렌더링 ← requirement 디자인 요구
* [ ] 8. 좌측 플레이어(2번)와 우측 플레이어(1번)의 초기 위치 사각형(도형)으로 렌더링 ← requirement 디자인 요구
* [ ] 9. 공(Ball) 오브젝트의 초기 위치 원형(도형)으로 화면 상단에 렌더링 ← requirement 디자인 요구

## Phase 4: 키보드 입력 및 플레이어 이동 제어
* [ ] 10. `keydown`, `keyup` 이벤트를 이용한 글로벌 키 상태 관리 객체(`keys`) 구현 ← requirement 기술 스택
* [ ] 11. 1번 플레이어 화살표 `←`, `→` 입력 시 x 좌표 변경 (화면 우측 절반+네트 경계 제한) ← requirement 세부 기능
* [ ] 12. 2번 플레이어 `A`, `D` 입력 시 x 좌표 변경 (화면 좌측 절반+네트 경계 제한) ← requirement 세부 기능

## Phase 5: 점프(연직 운동) 및 애니메이션 구현
* [ ] 13. 1번 플레이어 `↑` 입력 시 위쪽 가속도 부여 및 중력 상수를 이용한 점프/낙하(y 좌표) 물리 구현 ← requirement 세부 기능
* [ ] 14. 2번 플레이어 `W` 입력 시 위쪽 가속도 부여 및 중력 상수를 이용한 점프/낙하(y 좌표) 물리 구현 ← requirement 세부 기능
* [ ] 15. 플레이어가 바닥에 착지해 있고 좌우 이동 중일 때만 다리 번갈아 뛰기(2장 프레임 변수 토글) 상태 구현 ← requirement 세부 기능
* [ ] 16. 다리 프레임 변수 상태에 따라 플레이어 도형 하단에 다리 모양(간단한 사각형 분할 등) 교차 표현 ← requirement 세부 기능
* [ ] 17. 플레이어가 공중에 있거나 정지해 있을 때는 다리 애니메이션을 멈추고 기본 상태로 고정 ← requirement 세부 기능

## Phase 6: 공(Ball)의 기본 물리 및 네트 충돌
* [ ] 18. 공에 중력 가속도 및 x, y축 속도 변수를 적용하여 포물선으로 떨어지는 낙하 물리 구현 ← requirement 필수 기능
* [ ] 19. 공이 좌우 벽면 및 천장에 부딪혔을 때 속도를 반전시키는 튕김(반사) 물리 구현 ← requirement 세부 기능
* [ ] 20. 공이 중앙 네트(직사각형)와 충돌했는지 검사하는 AABB 충돌 알고리즘 구현 ← requirement 필수 기능
* [ ] 21. 네트 충돌 시 공의 x축 속도를 0으로 만들고 아래로 툭 떨어지게(y축 속도만 유지) 처리 ← requirement 세부 기능

## Phase 7: 플레이어 충돌 및 일반 반사
* [ ] 22. 공과 플레이어(사각형) 간의 충돌 감지 및 부딪힌 방향(위, 옆) 판정 로직 구현 ← requirement 필수 기능
* [ ] 23. 일반 충돌 시 부딪힌 위치/방향에 따라 공의 x, y 속도 벡터를 계산하여 튕겨 나가도록 구현 ← requirement 세부 기능

## Phase 8: 스파이크 시스템 구현
* [ ] 24. 공과 캐릭터가 충돌한 시점에 스파이크 키(`Enter` / `Z`)가 눌려 있는지 판정하는 조건문 추가 ← requirement 세부 기능
* [ ] 25. 2번(좌측) 플레이어가 `W`(위)와 함께 스파이크 시 공을 오른쪽 위 대각선 방향 고속 이동 처리 ← requirement 세부 기능
* [ ] 26. 2번(좌측) 플레이어가 `D`(우)와 함께 스파이크 시 공을 완만한 우측 고속 포물선 이동 처리 ← requirement 세부 기능
* [ ] 27. 1번(우측) 플레이어가 `↑`(위)와 함께 스파이크 시 공을 왼쪽 위 대각선 방향 고속 이동 처리 ← requirement 세부 기능
* [ ] 28. 1번(우측) 플레이어가 `←`(좌)와 함께 스파이크 시 공을 완만한 좌측 고속 포물선 이동 처리 ← requirement 세부 기능

## Phase 9: 득점 판정 및 라운드 리셋
* [ ] 29. 공이 바닥에 닿았을 때 네트 기준 왼쪽이면 우측 플레이어 스코어 +1 점수 증가 로직 구현 ← requirement 세부 기능
* [ ] 30. 공이 바닥에 닿았을 때 네트 기준 오른쪽이면 좌측 플레이어 스코어 +1 점수 증가 로직 구현 ← requirement 세부 기능
* [ ] 31. 득점 발생 시 점수 UI를 갱신하고, 공과 캐릭터 위치를 초기 서브 상태로 리셋하는 함수 구현 ← requirement 세부 기능

## Phase 10: 게임 종료 및 초기화 (최종 점검)
* [ ] 32. 매 득점 시 플레이어 점수가 5점에 도달했는지 확인하는 승리 조건 감지 루틴 추가 ← requirement 필수 기능
* [ ] 33. 5점 도달 시 게임 루프를 정지하고, 양쪽 승점을 0으로 초기화하는 데이터 리셋 구현 ← requirement 세부 기능
* [ ] 34. 스코어 초기화 완료 후 Canvas 화면을 지우고 다시 Phase 2의 '초기 화면(메인 메뉴)' 레이어를 노출 ← requirement 세부 기능
